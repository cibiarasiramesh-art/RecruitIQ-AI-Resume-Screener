# RecruitIQ вҖ” AI-Powered Resume Intelligence Platform

RecruitIQ analyses any resume against any job description, computes an ATS-readiness score, surfaces skill gaps with a learning plan, and generates a downloadable PDF report.

## вңЁ Features

- рҹ”җ **Authentication** вҖ” register, login, logout, hashed passwords, sessions, remember-me, route protection
- рҹ“Ө **Resume upload** вҖ” PDF & TXT, drag-and-drop or browse, file preview, validation
- рҹ“қ **Job description** вҖ” large text area, character counter, sample JD, clear button
- рҹ§  **Resume analysis** вҖ” programming languages, frameworks, libraries, databases, cloud, tools, soft skills
- рҹҺҜ **Scoring** вҖ” ATS match, skill, education, experience, overall + grade & seniority level
- рҹ“Ҡ **Visuals** вҖ” circular gauge, pie chart, bar chart, animated progress bars
- рҹҡҖ **Skill gap intelligence** вҖ” priority, time, impact, difficulty, docs / course / YouTube / practice links
- рҹ“қ **Improvement plan** вҖ” formatting, ATS keywords, portfolio, GitHub, LinkedIn, certifications, projects
- рҹ’ј **Career recommendations** вҖ” Frontend, Backend, Full Stack, Data, DevOps, UI/UX, Software Engineer
- рҹ•ҳ **History** вҖ” search, sort, view past reports, delete
- рҹ“„ **PDF reports** вҖ” branded, downloadable (client-side jsPDF + server-side fallback)
- рҹ‘Ө **Profile** вҖ” edit name/email, change password, upload avatar, delete account
- рҹҢ— **Dark/Light** glassmorphism UI, fully responsive

## рҹҡҖ Run locally

```bash
# 1. Clone
git clone https://github.com/<you>/RecruitIQ.git
cd RecruitIQ

# 2. Create & activate a virtual environment
python -m venv venv
# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Run
python app.py
```

Open <http://localhost:5000>.

## вҳҒпёҸ Deploy to Render

1. Push the repository to GitHub.
2. Go to <https://render.com> вҶ’ **New +** вҶ’ **Web Service**.
3. Connect your GitHub repo.
4. Render will detect `render.yaml` automatically. If not:
   - **Environment:** Python 3
   - **Build command:** `pip install -r requirements.txt`
   - **Start command:** `gunicorn app:app`
5. Click **Deploy**. That's it вҖ” no code changes required.

The `SECRET_KEY` is auto-generated by Render through `render.yaml`.

## рҹ”җ Google OAuth (Optional)

RecruitIQ supports Google OAuth for sign-in. By default, if `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` are not set, the app uses a demo login flow at `/auth/google/demo` that creates a placeholder user locally.

To enable real Google OAuth, set the following environment variables before running the app (or add them to your process manager / deployment platform):

```
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
GOOGLE_REDIRECT_URI=http://127.0.0.1:5000/auth/google/callback
```

Notes:
- The redirect URI above is the default used by the app. When creating OAuth credentials in Google Cloud Console, ensure this exact redirect URI is registered for your OAuth client.
- After setting these variables, the **Continue with Google** button will initiate a real Google sign-in flow.

If you prefer to keep the application fully offline, do not set the Google env vars вҖ” the demo flow will be used instead and no external requests will be made.

## рҹӣ  Offline Sentence-Transformer setup

RecruitIQ can use a Sentence-Transformers model for semantic resumeвҶ”JD matching. To keep the application fully offline, follow these steps **once** on a machine with internet access to download the model into a local folder:

1. Ensure dependencies are installed:

```bash
pip install -r requirements.txt
```

2. Run the setup script to download and save the model locally:

```bash
python scripts/setup_sentence_transformer.py
```

By default the script downloads `all-MiniLM-L6-v2` and saves it to `models/sentence_transformer/`.

3. (Optional) If you used a custom path, set the environment variable before running the app:

```bash
export SENTENCE_TRANSFORMER_LOCAL_DIR=models/sentence_transformer
# Windows (PowerShell)
$env:SENTENCE_TRANSFORMER_LOCAL_DIR = 'models/sentence_transformer'
```

Behavior:
- The application will only attempt to load the Sentence-Transformers model from the configured local directory (`SENTENCE_TRANSFORMER_LOCAL_DIR` or `models/sentence_transformer/`).
- If the local model is missing, the app logs a clear warning and continues using the local scikit-learn model as a fallback. It will NOT attempt to download the model at runtime.
- Do NOT commit the downloaded model files to git; add `models/` to `.gitignore` if desired.
